# static decrypt() → {GenericResponse|String}

aes解密
Parameters:
Name Type Description
args.encryptedData String 待解密数据
args.key String 加密时用的密钥
args.iv String aes IV字段。解密需要用到的字段。IV 是用于确保每次加密过程生成的密文是不同的，即使相同的明文也不会得到相同的密文。这对于增加安全性非常重要。
Since:
  • release-0.0.2
res - 通用返回
GenericResponse
res.data - 解密完成后的数据。如果解密失败，该字段会是一段无意义的乱码
String

# static encrypt() → {GenericResponse|String|String}

aes加密
Parameters:
Name Type Description
args.data String 待加密数据
args.key String 自定义任意字符串密钥
Since:
  • release-0.0.2
res - 通用返回
GenericResponse
res.iv - 解密需要用到的字段。IV 是用于确保每次加密过程生成的密文是不同的，即使相同的明文也不会得到相同的密文。这对于增加安全性非常重要。
String
res.encryptedData - 加密后的内容，解密的时候要用到
String
Example
const key = 'ktool-key'
const sourceData = 'hello world'
const enData = await kt.crypto.aes.encrypt({ key: key, data: sourceData })
// => 将 enData.encryptedData 和 enData.iv 发送给Bob即可。
